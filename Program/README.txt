Программа обрабатывает математическую модель, основанную на гиперграфе со следующем построением:
Два множества вершин типа L и R, с одинаковой размерностью.
Множество вершин типа I с другой размерностью, у всех вершин типа I есть значение I[i]
Математическая модель должна построить группу ребер с однозначно сопоставленными вершинами L и R, и сопоставленной им вершиной I, i'тая вершина из Множества I в одной группе ребер может встречаться I[i] количество раз. Вершины L и R могут встречаться лишь один раз.
Модель n-критериальная, это означает что одинаковым ребрам в разных критериях могут быть назначены n разны[ значений.

Инструкция:
Для начала необходимо запустить файл Intro.exe, и выполнять все прописанные программой инструкции.
1. Вводится количество критериев задачи (Одно число)
2. Необходимо ввести количество вершин типа L и R (одно число)
3. Количество вершин типа I
4. Далее по одному числу через пробел вводим веса тех вершин, которые указаны в диалоговом окне (Все вершины считаются с нуля) Вершины вводятся поочереди для каждого критерия.
5. Далее, можно отредактировать введенные выше данные нажав клавишу 'r'. (Все типы вершин считаются с нуля, но(!) критерии считаются с единицы(1,2,3...)) Чтобы продолжить, нужно нажать клавишу 'n'.
6. Далее необходимо ввести значения для вершин типа I (Количество возможных включений i'той вершины в одно группу ребер)
7. С помощью клавиши 'r' можно отредактировать эти данные или сохранить их с помощью клавиши 's'. Данные сохраняются в input.txt

(!!!)Важно: Сумма всех значений вершин типа I, должна превышать количество вершин типа L и R.

Данные в файле input.txt имеют такой вид:
<количество критериев>
<количество вершин типа L и R> <Количество вершин типа I>
{Множество значений вершин типа I}
{Множество весов для всех возможных ребер}

После того как в файл input.txt записались необходимые значения, можно запускать программу Outro.exe.
Для работы может потребоваться некоторое(иногда довольно продолжительное время.)
В идеале на выходе программа должна представить множество несравнимых максимальных векторов. Каждый вектор - это сумма ребер по одному из критериев. Размер вектора равен количеству критериев. (Одинаковые вектора возможны). Итоговый результат записывается в файл result.txt
Все возможные переборы групп ребер и их значения по критериям можно посмотреть в файле log.txt

(!!!!!!)P.S. Существует большая проблема со скоростью работы программы. Проблема эта зависит напрямую от количества всех вершин, но тут существует сложность. Не так важно количество вершин типа L и R (хотя казалось бы именно от них зависит количество ребер в каждой группе), сколько важно количество вершин типа I, и их значения. При условном I[i] с большим значением вариантов перебора становится в разы больше. Возможно это связано с особенностями перебора.

Т.е. Будут попадаться такие наборы, при одинаковом количестве вершин, которые будут работать абсолютно разное количество времени, и чем больше вершин тем эти наборы вероятней.
(!!!)Я тестировал программу в среднем с 5 вершинами типа L и R и меньшем количестве вершин I

Решение: Перебор оптимизировался несколько раз, но возможно проблема с самой концепцией перебора и ее необходимо сменить, если это невозможно, может стоить рассмотреть вариант перехода с перебора на более сложный алгоритм.
